# Community Launch Plan

Cloud Recall Lab should stay free, useful, and trustworthy. The community plan is designed to grow the project without adding accounts, tracking, or copied exam material.

## Launch Week

- Create the public GitHub repo and enable GitHub Pages.
- Pin a comment on the LinkedIn launch post with the live app, repo, and Buy Me a Coffee links.
- Ask for specific feedback: mobile bugs, confusing explanations, outdated AWS wording, and original question ideas.
- Enable GitHub Discussions after the repo is public so learners can ask Q&A without every question becoming a bug.
- Cross-post carefully to r/AWSCertifications only if the subreddit rules allow free tools.
- Write a short Dev.to or Hashnode post: "How I Built a Free AWS Exam Prep Tool with Zero Dependencies."
- Ask early users to comment if the tool helped them study, but do not imply it guarantees a pass.

## GitHub Hygiene

- Keep issue templates enabled for bugs, feature requests, and question corrections.
- Require every question PR to confirm it is original and not copied from an exam dump, paid course, or protected material.
- Respond to issues with labels such as `bug`, `question-correction`, `accessibility`, `mobile`, and `good first issue`.
- Add a star badge to the README after the repo exists.
- Use GitHub Releases for public milestones such as v1 launch, question-pack updates, and PWA fixes.

## Good Community Asks

- "Report anything that feels outdated compared with current AWS docs."
- "Suggest original scenarios that test a confusing service difference."
- "Share screenshots of mobile layout problems."
- "Tell me which section felt weakest after using the progress dashboard."

## Defer Until There Is Traction

- Discord server: wait until there is steady activity or roughly 100+ stars.
- Product Hunt: wait until the hosted app, README, screenshots, and issue templates are stable.
- Privacy-friendly analytics: optional later, and only with a clear notice.
- Full translated question packs: only after native or highly fluent reviewer QA.
